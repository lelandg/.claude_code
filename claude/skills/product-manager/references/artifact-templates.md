## Product Requirements Document (PRD)

```markdown
# PRD: [Product/Feature Name]

**Author:** [Name] | **Date:** [Date] | **Status:** Draft / In Review / Approved

## Problem Statement
[What user problem does this solve? Include data/evidence.]

## Goals & Success Metrics
| Goal | Metric | Target |
|------|--------|--------|
| [Goal 1] | [KPI] | [Target value] |
| [Goal 2] | [KPI] | [Target value] |

## Target Users
[Primary and secondary user personas. Link to persona docs if available.]

## User Stories
- As a [user type], I want [action] so that [benefit]
- As a [user type], I want [action] so that [benefit]

## Requirements

### Must Have (P0)
- [ ] [Requirement 1]
- [ ] [Requirement 2]

### Should Have (P1)
- [ ] [Requirement 1]

### Nice to Have (P2)
- [ ] [Requirement 1]

## Technical Considerations
[Architecture constraints, dependencies, integrations, performance requirements]

## Design
[Link to mockups/wireframes, key UX decisions]

## Timeline
| Milestone | Target Date |
|-----------|------------|
| Design complete | [Date] |
| Dev complete | [Date] |
| QA/Testing | [Date] |
| Launch | [Date] |

## Risks & Mitigations
| Risk | Impact | Mitigation |
|------|--------|------------|
| [Risk 1] | [H/M/L] | [Plan] |

## Open Questions
- [Question 1]
```

## Product Analysis Report

Use this template when analyzing an existing product's health and strategy.

```markdown
# Product Analysis: [Product Name]

**Date:** [Date] | **Analyst:** [Name]

## Product Overview
[What it does, who it's for, core value proposition]

## Life Cycle Stage Assessment
**Current stage:** [Development / Introduction / Growth / Maturity / Decline]
**Evidence:** [Data points supporting this assessment]
**Recommended focus:** [Strategic priorities for this stage]

## Market Position
- **Target market:** [Description]
- **Market share:** [If known]
- **Competitive position:** [Leader / Challenger / Niche / Follower]

## Metrics Health Check
| Category | Metric | Current | Trend | Assessment |
|----------|--------|---------|-------|------------|
| Acquisition | [Metric] | [Value] | [Up/Down/Flat] | [Good/Needs Attention/Critical] |
| Engagement | [Metric] | [Value] | [Trend] | [Assessment] |
| Retention | [Metric] | [Value] | [Trend] | [Assessment] |
| Revenue | [Metric] | [Value] | [Trend] | [Assessment] |

## SWOT Analysis
| Strengths | Weaknesses |
|-----------|------------|
| - [S1] | - [W1] |

| Opportunities | Threats |
|--------------|---------|
| - [O1] | - [T1] |

## Recommendations
1. [Priority 1 action with rationale]
2. [Priority 2 action with rationale]
3. [Priority 3 action with rationale]
```
